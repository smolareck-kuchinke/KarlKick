## Iteration 3 – Client-seitiges Routing

### Ziel
Einführung eines client-seitigen Routings mit React Router, sodass zwischen den Hauptseiten der Anwendung navigiert werden kann.

### Verwendeter Prompt
Siehe `prompts/04-routing.md`.

### Umsetzung
- React Router in das Frontend integriert.
- Navigation zwischen den Seiten Dashboard, Tipps, Rangliste, Freunde und Profil umgesetzt.
- Das bestehende Dashboard unverändert beibehalten.
- Für die übrigen Seiten zunächst Platzhalter erstellt, die in späteren Iterationen mit Funktionalität erweitert werden.

### Ergebnis
Die Navigation innerhalb der Anwendung funktioniert vollständig über das client-seitige Routing. Ein Wechsel zwischen den einzelnen Seiten ist ohne Neuladen der Anwendung möglich.

### Screenshots
Siehe `screens/iterations/iteration-04-routing/`.

### Hinweise
- Keine Backend-Anbindung.
- Keine Benutzerauthentifizierung.
- Keine Geschäftslogik implementiert.
- Die Platzhalterseiten dienen als Grundlage für die folgenden Entwicklungsschritte.



## Iteration 4 – Tippabgabe

### Verwendetes AI-Tool

Claude Code

### Prompt

Siehe `prompts/05-predictions.md`.

### Ergebnis

- Seite „Predictions“ vollständig umgesetzt.
- Fünf Beispielspiele mit Dummy-Daten integriert.
- Für jedes Spiel können Tipp-Ergebnisse eingegeben werden.

ohne bereits ein Backend oder eine Datenbank einzubinden. Dadurch bleibt die Entwicklung weiterhin in kleinen, nachvollziehbaren Schritten.

Implementierung einer Ranglisten-Seite, auf der die aktuellen Punktestände der Spieler dargestellt werden. Die Seite soll das bestehende Design wiederverwenden und zunächst ausschließlich mit Dummy-Daten arbeiten.

### Verwendetes AI-Tool

Claude Code

### Prompt

Siehe `prompts/06-leaderboard.md`.

### Ergebnis

- Ranglisten-Seite implementiert.
- Top-10-Spieler mit Rang, Name und Punktestand dargestellt.
- Trendanzeige (Aufstieg, Abstieg oder unverändert) integriert.

Die Wiederverwendung bestehender Komponenten reduziert redundanten Code und erleichtert die spätere Erweiterung der Anwendung. Die Rangliste arbeitet derzeit ausschließlich mit Dummy-Daten und bildet die Grundlage für eine spätere Anbindung an das Backend.

## Iteration 6 – Freunde

### Ziel

Implementierung einer Freundesübersicht, in der die Benutzer ihre Freunde, die aktuelle Tippspiel-Liga sowie einen Einladungscode einsehen können. Die Seite soll den sozialen Aspekt der Anwendung darstellen und als Grundlage für spätere Mehrspielerfunktionen dienen.

### Verwendetes AI-Tool

Claude Code

### Prompt

Siehe `prompts/07-friends.md`.

### Ergebnis

- Freundesübersicht implementiert.
- Fünf Beispiel-Freunde mit Dummy-Daten dargestellt.
- Für jeden Freund werden Name, Lieblingsverein sowie Online-/Offline-Status angezeigt.
- Private Tippspiel-Liga „Karl's Bundesliga“ integriert.
- Einladungscode „KARL2026“ angezeigt.
- Schaltfläche „Invite Friend“ ergänzt (derzeit ohne Funktion).
- Bestehende React-Komponenten und Designrichtlinien konsequent wiederverwendet.
- Keine Backend-Anbindung.
- Keine Benutzerauthentifizierung.

### Eigene Erkenntnisse

Durch die Erweiterung der Anwendung um eine soziale Komponente wirkt KarlKick bereits deutlich realistischer. Die Wiederverwendung bestehender Komponenten und Designmuster sorgte dafür, dass sich die neue Seite nahtlos in die vorhandene Benutzeroberfläche integriert. Die derzeit verwendeten Dummy-Daten können in einer späteren Iteration problemlos durch Daten aus dem Backend ersetzt werden.

### Screenshots

Siehe `screens/iterations/iteration-06-friends/`.
