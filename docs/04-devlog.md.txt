## Iteration 2 – Dashboard

### Ziel
Implementierung eines ersten Dashboard-Prototyps auf Basis des mit Google Stitch entworfenen UI-Designs.

### Verwendetes AI-Tool
Claude Code

### Prompt
Implement ONLY the Dashboard page using reusable React components and dummy data.

### Ergebnis
- Dashboard-Seite erstellt
- Wiederverwendbare React-Komponenten angelegt
- Dummy-Daten für Spieltag, Spiele und Rangliste verwendet
- Modernes Dark Theme entsprechend des Google-Stitch-Entwurfs umgesetzt
- Keine Backend-Anbindung
- Keine Login-Funktion
- Keine Businesslogik

### Eigene Erkenntnisse
Claude Code konnte aus einem präzisen Prompt eine gut strukturierte React-Oberfläche erzeugen. Durch die Aufteilung in einzelne Komponenten bleibt die Anwendung leicht erweiterbar. Das Dashboard entspricht bereits weitgehend dem zuvor entworfenen UI-Prototyp.



## Iteration 3 – Client-seitiges Routing

### Ziel
Einführung eines client-seitigen Routings mit React Router, sodass zwischen den Hauptseiten der Anwendung navigiert werden kann.

### Verwendeter Prompt
Siehe `prompts/04-routing.md`.

### Umsetzung
- React Router in das Frontend integriert.
- Navigation zwischen den Seiten Dashboard, Tipps, Rangliste, Freunde und Profil umgesetzt.
- Das bestehende Dashboard unverändert beibehalten.
- Für die übrigen Seiten zunächst Platzhalter erstellt, die in späteren Iterationen mit Funktionalität erweitert werden.

### Ergebnis
Die Navigation innerhalb der Anwendung funktioniert vollständig über das client-seitige Routing. Ein Wechsel zwischen den einzelnen Seiten ist ohne Neuladen der Anwendung möglich.

### Screenshots
Siehe `screens/iterations/iteration-04-routing/`.

### Hinweise
- Keine Backend-Anbindung.
- Keine Benutzerauthentifizierung.
- Keine Geschäftslogik implementiert.
- Die Platzhalterseiten dienen als Grundlage für die folgenden Entwicklungsschritte.



## Iteration 4 – Tippabgabe

### Ziel

Implementierung der ersten interaktiven Funktion der Anwendung. Benutzer sollen Spielergebnisse eingeben und ihre Tipps lokal speichern können.

### Verwendetes AI-Tool

Claude Code

### Prompt

Siehe `prompts/05-predictions.md`.

### Ergebnis

- Seite „Predictions“ vollständig umgesetzt.
- Fünf Beispielspiele mit Dummy-Daten integriert.
- Für jedes Spiel können Tipp-Ergebnisse eingegeben werden.
- Tipps werden lokal im React State gespeichert.
- Nach dem Speichern erscheint eine Erfolgsmeldung.
- Keine Backend-Anbindung und keine Datenpersistenz.

### Eigene Erkenntnisse

Mit dieser Iteration besitzt KarlKick erstmals eine echte Benutzerinteraktion. Durch die Verwendung des React State konnte die Funktionalität umgesetzt werden, ohne bereits ein Backend oder eine Datenbank einzubinden. Dadurch bleibt die Entwicklung weiterhin in kleinen, nachvollziehbaren Schritten.
